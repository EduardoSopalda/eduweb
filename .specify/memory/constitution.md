# eduweb Constitution

Guiding principles for the Eduardo Sopalda portfolio (eduardosopalda.com).

## Project identity
A personal portfolio that is also a creative experiment: cinematic web design, surrealist butcher's shop concept, human centered storytelling, built in open collaboration with AI. The site documents the journey from idea to launch. Playful on the surface, disciplined underneath.

## Principles

### 1. Spec first
No code without a spec. Every change begins as a short written specification, reviewed before anything is implemented. The spec is the source of truth. If a request is ambiguous, write the spec until it is not.

### 2. Simplicity is a feature
The codebase must stay readable and maintainable by someone still leveling up as an engineer. Prefer the simplest solution that works. Avoid dependencies, abstractions, and frameworks that are not earning their place. Clear beats clever.

### 3. Creative intent is a requirement
The surrealist concept, the cinematic tone, and the voice are first class constraints, not decoration. A change that ships correct code but breaks the feeling has failed its spec. Visual identity and narrative belong in the spec alongside the technical requirements.

### 4. Human centered and accessible
The visitor comes first. Semantic markup, keyboard navigation, sufficient contrast, meaningful alt text, and sensible focus order are baseline. The site must work and feel right on mobile before it is done.

### 5. Reviewable by design
Work happens on branches, never directly on main. Every change is reviewable before merge. The phase gates (specify, plan, tasks, implement) are honored in order, with a human review between each. Small, legible commits over large opaque ones.

### 6. Performance and resilience
Fast to load. Graceful when something is missing or slow. Degrades cleanly rather than breaking. Speed is part of the experience, not a later tuning step.

### 7. Document the journey
Capture decisions, not just outcomes. When something fails or a direction is abandoned, write down why. The repo is a record of how the thing was made.

## Governing stance
Do not mandate. Illuminate. This constitution exists to make the right thing the easy thing, not to police. When a rule and a good outcome conflict, surface the tension in the spec, decide deliberately, then update this document so the next decision is easier.
